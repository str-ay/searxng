# Architecture Decision Records

## ADR-001: Таймауты поисковых движков SearXNG

### Дата
2026-03-13

### Статус
Принято

### Контекст
SearXNG агрегирует результаты из множества поисковых движков параллельно. Время ответа определяется самым медленным активным движком. Возник вопрос: какие таймауты установить?

### Рассмотренные варианты

**Вариант A — Минимальные таймауты (1-2 секунды)**
- ✅ Плюс: максимально быстрый ответ
- ⚠️ Риск: потеря результатов от медленных ценных источников

**Вариант B — Дифференцированные таймауты**
- ✅ Плюс: баланс скорости и качества
- ⚠️ Риск: сложность конфигурации

**Вариант C — Максимальные таймауты (5-6 секунд)**
- ✅ Плюс: максимум результатов
- ⚠️ Риск: медленный ответ для простых запросов

### Решение
Принято **Вариант B** — дифференцированные таймауты по категориям движков.

| Категория | Таймаут | Примеры |
|-----------|---------|---------|
| Быстрые поисковики | 4.0s | google, brave |
| Академические источники | 5-6s | arxiv, semantic_scholar, crossref |
| Сообщества | 3-4s | hackernews, reddit |
| Репозитории кода | 4-5s | github, gitlab |
| Пакетные менеджеры | 3-4s | npm, pypi, crates |
| Проблемные | отключены | startpage |

### Обоснование

1. **Таймаут не ограничивает быстрые движки** — google/brave отвечают за 0.5-1s, таймаут 4s на них не влияет

2. **Медленные источники ценны** — академические статьи и репозитории требуют времени, но дают качественные результаты

3. **Скорость ответа ~1.3s** — определяется быстрыми движками, академические отвечают параллельно

4. **startpage отключён** — регулярные таймауты, нет уникальных результатов

### Последствия

**Позитивные:**
- Качественные академические результаты включаются в ответ
- Скорость осталась высокой (~1.3s)
- Стабильность (убран проблемный движок)

**Негативные:**
- Для запросов где нужны только быстрые движки, всё равно ждём академические (но они отвечают параллельно)
